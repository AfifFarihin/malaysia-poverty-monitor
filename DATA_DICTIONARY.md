# Data Dictionary

This dictionary covers the active Malaysia state-level data used by the repository. All paths are relative to the repository root.

## Active Data Files

### `data/raw/dosm/hh_poverty_state.csv`

Untouched official DOSM state-level poverty data retained from the OpenDOSM `hh_poverty_state` dataset.

Columns: `date`, `state`, `poverty_absolute`, `poverty_hardcore`, `poverty_relative`.

### `data/interim/dosm_poverty_state_clean.csv`

Cleaned DOSM poverty labels generated by `notebooks/00_dosm_raw_to_labels.ipynb`. This file converts `date` to `year`, maps DOSM state names to the project `state_key`, and filters to the 10 survey years used in the project.

Columns: `state_key`, `state_raw`, `year`, `poverty_absolute`, `poverty_hardcore`, `poverty_relative`.

### `data/state_year_panel_modelready_2002_2022.csv`

Final Malaysia state-year modelling panel. This is not raw data. It is the stable downstream panel that joins cleaned DOSM poverty labels with derived satellite/geospatial state features.

| Field group | Description |
|---|---|
| Identifiers | `state`, `state_key`, `year` identify the 16 Malaysia states/federal territories across official DOSM survey years. |
| Targets | `poverty_absolute`, `poverty_relative`, and `poverty_hardcore` record official poverty rates where available. `poverty_absolute` and `poverty_hardcore` each have one missing row: W.P. Labuan 2002. Missing target labels are excluded from supervised evaluation rather than imputed as ground truth. |
| Nighttime lights | DMSP/VIIRS-derived brightness features, including raw, ranked, and year-standardised measures. |
| Vegetation | EVI and NDVI features used as agricultural/vegetation proxy signals. |
| Urban/population | Urban fraction and WorldPop-derived population features. |
| Topography | Elevation and slope features from SRTM-derived terrain variables. |

Active model selection uses `poverty_absolute` as the primary target. Relative poverty remains in the comparison because it shows an important negative result: year-only dynamics outperform satellite/geospatial features for that target.

### `data/state_model_predictions.csv`

Generated selected-model prediction artifact created by `notebooks/03_model_training_evaluation.ipynb`. This file is not raw data and is not an external source file. It contains selected-model predictions by state-year for dashboard and report consumption. After the target-handling repair, the absolute-poverty prediction column may be missing for W.P. Labuan 2002 because that row has no observed absolute-poverty target for LOSO evaluation.

Columns: `state_key`, `year`, `predicted_poverty_absolute`, `predicted_poverty_relative`.

### `data/boundaries/malaysia_adm1_geoboundaries.geojson`

Simplified Malaysia ADM1 boundaries from the geoBoundaries `gbOpen` release, used by the dashboard map. The boundary file is provided under the Open Data Commons Open Database License 1.0; see `DATA_LICENSES.md`.

## Active Output Tables

### `outputs/metrics/malaysia_state_model_comparison.csv`

State-level LOSO model comparison table for Random Forest, XGBoost, Ridge, ElasticNet, and year-only baselines. Absolute-poverty benchmark rows use 157 observed target rows; relative-poverty benchmark rows use 158 rows.

Preprocessing, including median imputation and scaling where applicable, is fitted inside each LOSO training fold. The primary absolute-poverty model is chosen only from the fixed `sensor_safe` feature policy by lowest LOSO MAE, then higher Spearman and model name as deterministic tie-breakers.

Required reporting fields include:

- `target`
- `model`
- `loso_mae_pp`
- `loso_rmse`
- `loso_r2`
- `loso_spearman`

### `outputs/metrics/malaysia_state_loso_fold_predictions.csv`

Fold-level state-year predictions used for LOSO diagnostics, dashboard trend comparisons, and state-level error inspection.

### `outputs/metrics/malaysia_state_uncertainty_intervals.csv`

Bootstrap/fold uncertainty intervals used to avoid overclaiming small performance differences between RF and XGBoost.

### `outputs/metrics/malaysia_state_temporal_backtest.csv`

Rolling-origin predictions for the selected absolute-poverty model. Every test row is predicted by a model trained only on strictly earlier survey years. This is a secondary robustness check rather than an independent model-selection dataset.

### `outputs/metrics/malaysia_state_rf_shap_poverty_absolute.csv`

Ranked RF SHAP feature importance table for the selected state-level absolute-poverty Random Forest model.
